Privileged Permissions

MAY 03, 2023 - TheHitMan7

Android Permissions

Android permissions provide controls that increase user awareness and limit an app’s access to sensitive data. Privileged apps are system apps that are located in a priv-app directory on one of the system image partitions. Configuring permissions on Android 8.0 and lower includes allowlisting, without which privileged apps are disabled, even if they’re in the priv-app path. On Android 9 and higher, a device that tries to use apps that aren’t properly allowlisted won’t boot. Implementors must explicitly grant or deny all privileged permissions. 

Some system apps are more system than others.
Missing Permissions

When permissions are missing from system configuration XML files in the etc/permissions directory, the device stuck in an endless bootloop. All violations must be addressed by adding the missing permissions to the appropriate allowlists. You must explicitly either allow or deny all privileged permissions.

BiTGApps Installation

Privileged permissions are already defined for all priv-app included in BiTGApps installer. However it is not possible for us to test installer on every supported Android version. Facing bootloop even with successful BiTGApps installation then this might be the case.

Troubleshoot Installation

You need to grab logcat, when device stuck in an endless bootloop. Follow below command: 

adb logcat > bootloop.log

When troubleshooting, make sure that you have just ROM and BiTGApps installed.

Avoid using Custom Kernel, Magisk or other modifications.
