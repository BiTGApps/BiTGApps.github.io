BiTGApps

Advance Features

Take BiTGApps installation to next level. Flash BiTGApps from booted device using either Magisk or KernelSU with a possible factory reset to make it exactly like Google Apps installed before first boot.

Commonize

This new feature, let you flash BiTGApps from booted device using either Magisk or KernelSU. But wait, this is not a systemless installation or some module. This is exactly similar to, how you install BiTGApps from recovery like TWRP, Lineage Recovery, etc and everything goes directly into System. So when you install BiTGApps from Magisk or KernelSU, everything installs directly into System. Factory reset is also possible to make it exactly like Google Apps installed before first boot.

Experimental

Continuing from above, when you pass required property using configuration file before installation, then it will enable systemless functionality and install BiTGApps as Magisk or KernelSU module. There aren't different builds for different use case. There is just one BiTGApps build to rule them all. The disadvantage of going systemless is, you cannot factory reset to make it exactly like Google Apps installed before first boot, which can create some issues but it also depends on your use case.

Systemless Advantage

Even with low system space, Google Apps Package installation is possible. You can easily remove GApps installation.
